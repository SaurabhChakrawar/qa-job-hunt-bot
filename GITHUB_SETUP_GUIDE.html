Upload Your Bot Files to GitHub
From your Mac Terminal

First, tell Git who you are (one time only):

git config --global user.name "Your Name"
git config --global user.email "[email protected]"

Navigate to your downloaded bot folder:

The folder you downloaded is called job-automation-free. In Terminal:

# Replace with actual path where you downloaded it
# Example if it's in Downloads folder:
cd ~/Downloads/job-automation-free

Initialize and push to GitHub:

git init
git add .
git commit -m "Initial commit - QA Job Hunt Bot"
git branch -M main

git push -u origin main
üí° Replace YOUR_USERNAME with your actual GitHub username. You can find the exact URL on your GitHub repo page ‚Äî click the green "Code" button and copy the HTTPS URL.

GitHub will ask for your username and password. For the password, use a Personal Access Token (not your GitHub password):

  1. GitHub ‚Üí Settings ‚Üí Developer Settings ‚Üí Personal access tokens ‚Üí Tokens (classic)
  2. Click "Generate new token (classic)"
  3. Check the "repo" checkbox
  4. Click Generate ‚Üí Copy the token ‚Üí use it as your password in Terminal

Add Secrets to GitHub
This is where your API keys are stored safely ‚Äî GitHub encrypts them

Go to your GitHub repo ‚Üí Settings ‚Üí Secrets and variables ‚Üí Actions ‚Üí "New repository secret"

Add these 4 secrets one by one:

GEMINI_API_KEY
Your Gemini API key from Step 5 Example: AIzaSyXXXXXXXXXXXXXXXXXXXX
GMAIL_ADDRESS
Your Gmail address (report will be sent here) Example: [email protected]
GMAIL_APP_PASSWORD
Your 16-character Gmail App Password from Step 6 Example: abcdefghijklmnop (no spaces)
LINKEDIN_EMAIL
Your LinkedIn login email (for scraping jobs) Example: [email protected]
LINKEDIN_PASSWORD
Your LinkedIn password Used only for scraping job listings. Auto-apply is disabled.
RESUME_PROFILE_JSON
Your resume profile as JSON ‚Äî see instructions below ‚Üì This is the config/resume_profile.json content

How to get RESUME_PROFILE_JSON:

Run this on your Mac Terminal first (needs Gemini key set in local config.json):

cd ~/Downloads/job-automation-free
pip install -r requirements.txt
pip install PyMuPDF

# Set your Gemini key temporarily in config/config.json
# Then run:
python main.py --parse-resume /path/to/your_resume.pdf

This creates config/resume_profile.json. Then copy its entire contents:

cat config/resume_profile.json

Copy everything that appears ‚Üí paste it as the value for RESUME_PROFILE_JSON secret on GitHub.

üí° No resume PDF yet? Use the sample file: copy the contents of config/resume_profile.sample.json and manually edit it with your real skills. Then paste that as the secret.

Troubleshooting
If something goes wrong

‚ùå Workflow fails (red X)

Click on the failed run ‚Üí click on the failed step ‚Üí read the error message. Most common causes:

  • Wrong Gemini key ‚Üí Double-check GEMINI_API_KEY secret has no extra spaces
  • Email not sending ‚Üí Make sure GMAIL_APP_PASSWORD has no spaces (remove spaces from the 16-char password)
  • RESUME_PROFILE_JSON invalid ‚Üí Make sure you copied the entire JSON including the opening {'{'}{'}'} braces

‚ùå No email received

  • Check spam/junk folder
  • Check Gmail "All Mail" folder
  • Verify GMAIL_ADDRESS and GMAIL_APP_PASSWORD secrets are correct

‚ùå 0 jobs found

  • LinkedIn may be blocking ‚Äî that's okay, Remotive/WWR/Himalayas will still work
  • You'll still get jobs from 4+ other sources

python main.py --parse-resume /new/resume.pdf
